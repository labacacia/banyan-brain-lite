# Banyan Brain Lite 1.1.0 发布说明

Banyan Brain Lite 1.1.0 是 post-Wave GA 版本。它延续 1.0.0 的单节点、SQLite-backed Lite 边界，并补上首个稳定版之后完成的知识包、observability 和依赖加固工作。

## 发布范围

1.1.0 只对应 **Lite** 版本。

它仍然适合：

- 本地 agent memory；
- offline-first memory node；
- 小型单节点部署；
- demo 和集成测试；
- 需要 SQLite 持久记忆的嵌入式 agent runtime。

多租户 fleet topology、Ent 风格 policy governance、外部对象存储和云端协同仍属于 Pro/Ent 范围。

## 重点变化

### 知识包

- `.banyanpack` v2 manifest 和迁移支持。
- Ed25519 pack signing 与 verification。
- 通过 NID/CA 链建立 pack mount trust。
- In-pack embeddings 与 cosine recall，让 Lite 可以搜索 mounted packs，而不必把所有记录导入 live memory store。
- Pack version management：pin、upgrade、rollback 和 active-version switching。

### 检索和打包

- Lite recall 可以合并 live memory hits 和 mounted knowledge-pack vector hits。
- `Banyan.Embedders` 与 `Banyan.Embedders.Onnx` 拆分，让 CLI 和基础 library package 在不需要 ONNX 时保持更轻。
- public distribution 中恢复 ONNX project files，fresh clone 可以正常 build。

### Observability 和审计

- Lite OpenTelemetry foundation。
- 给本地 operator 使用的 memory-operation metrics。
- write、update、forget 操作的 tamper-evident audit records。

### 依赖加固

- NPS packages 对齐到 `1.0.0-alpha.12`。
- 显式 pin `MessagePack` 到 `3.1.7`，避免 NPS 传递恢复有漏洞的 3.0.x 版本。
- 显式 pin `SQLitePCLRaw.bundle_e_sqlite3` 到 `3.0.3`，避开有漏洞的 bundled SQLite native library。

## 安装

```bash
dotnet tool install -g Banyan.Cli --version 1.1.0
```

本地分发包也同步产出：

- Windows：`banyan-lite-1.1.0-setup.exe`，per-user NSIS installer，会写入
  `uninstall.exe` 和 Add/Remove Programs 卸载入口。
- Debian/Ubuntu：`banyan-lite_1.1.0_amd64.deb`，安装后提供
  `/usr/bin/banyan`。

## 升级

```bash
dotnet tool update -g Banyan.Cli --version 1.1.0
```

升级后验证 tool，并初始化缺失的本地资源：

```bash
banyan --version
banyan embedder download
```

生产 Lite 节点升级前，请先备份 SQLite data directory。

## 兼容性

- Target framework：`net10.0`。
- Package version：`1.1.0`。
- Storage：SQLite。
- License：Apache-2.0。
