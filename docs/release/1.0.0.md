# Banyan Brain Lite 1.0.0 Release Notes

Banyan Brain Lite 1.0.0 is the first stable Lite release. It packages the local/offline Memory Node distribution as a single .NET tool with SQLite storage, embedded NIP Mini-CA, OLS/OIDC operator identity, NID authentication, MCP support, and hybrid retrieval.

## Release scope

1.0.0 is the **Lite** edition only.

It is intended for:

- local agent memory;
- offline-first memory nodes;
- small single-node deployments;
- demos and integration testing;
- embedded agent runtimes that need SQLite-backed durable memory.

It is not the Pro or Ent distribution. Multi-tenant scope enforcement, external CA-first deployment, shared memory pools, Anchor Node ingress, Vector Proxy, Bridge Node adapters, orchestration, audit chain, and K-of-N quorum remain Pro/Ent responsibilities.

## Highlights

### Memory and retrieval

- Event-sourced memory write/update/forget log.
- Current snapshot table for fast reads.
- Hybrid search using BM25 / FTS5, ONNX embeddings, and RRF fusion.
- Optional `sqlite-vec` ANN acceleration with in-memory cosine fallback.
- Namespace column retained for forward compatibility with Pro, while Lite remains single-node and single-tenant by default.

### Protocol surface

- NPS-3 Memory Node host through `NPS.NWP`.
- `/.nwm` Neural Web Manifest endpoint.
- `/.schema` Memory Node schema endpoint.
- `POST /api/memory/query` NWP query endpoint.
- NCP anchor digest and token estimate metadata in query responses where supported.

### Identity and trust

- Embedded Ed25519 NIP Mini-CA.
- SQLite-backed CA ledger.
- Agent certificate issue / verify / revoke commands.
- NPS-3 §8 compatible CA HTTP routes.
- Remote CA verification via `--trusted-issuer` and `--ocsp-url`.
- NID middleware with `anonymous-allowed`, `writes-required`, and `all-required` modes.
- OLS/OIDC operator identity, first-run admin setup, login, JWT, and SQLite identity stores.

### Tooling and integrations

- `banyan` single-binary CLI as a .NET tool.
- Web UI for memory, agents, identity setup, and CA operations.
- `banyan mcp` stdio MCP server.
- Streamable HTTP MCP endpoint at `/mcp` when running `banyan web`.
- Offline embedder download workflow.

## Install

```bash
dotnet tool install -g Banyan.Cli --version 1.0.0
```

## Upgrade

```bash
dotnet tool update -g Banyan.Cli --version 1.0.0
```

After upgrading, verify the tool and initialize any missing local assets:

```bash
banyan --version
banyan embedder download
```

## First-run checklist

```bash
export BANYAN_NIP_CA_PASSPHRASE='your-passphrase'
banyan ca init
banyan init --admin-username admin --admin-password 'change-me-now'
banyan agent issue --id summarizer-01 --cap memory.read,memory.write \
  --key-out ~/.banyan/agents/summarizer-01.key
export BANYAN_EMBEDDER=onnx
banyan web
```

Open `http://localhost:5180` and sign in with the admin account.

## Production notes for Lite

- Use `--nid-auth writes-required` for the common production posture: public reads, authenticated writes.
- Use `--nid-auth all-required` when both reads and writes must be authenticated.
- Keep `BANYAN_NIP_CA_PASSPHRASE` outside shell history and service unit files when possible; prefer environment injection from the host secret manager.
- Back up the SQLite data directory before upgrades and before CA key rotation.
- If using a remote CA, pin the trusted issuer with `--trusted-issuer` and keep the OCSP endpoint reachable.
- Treat Lite as a single-node deployment. For tenant isolation or fleet topology, use the Pro/Ent roadmap rather than overloading Lite.
