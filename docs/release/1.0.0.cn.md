# Banyan Brain Lite 1.0.0 发布说明

Banyan Brain Lite 1.0.0 是 Lite 的第一个稳定发布版。它把本地/离线 Memory Node 打包成一个 .NET tool，包含 SQLite 存储、内置 NIP Mini-CA、OLS/OIDC 管理员身份、NID 鉴权、MCP 支持和混合检索。

## 发布范围

1.0.0 只对应 **Lite** 版本。

适用场景：

- 本地 agent memory；
- 离线优先 memory node；
- 小型单节点部署；
- demo 和集成测试；
- 需要 SQLite 持久记忆的嵌入式 agent runtime。

它不是 Pro 或 Ent 版本。多租户 scope enforcement、外置 CA-first 部署、共享记忆池、Anchor Node ingress、Vector Proxy、Bridge Node adapters、编排、审计链、K-of-N quorum 仍属于 Pro/Ent 范围。

## 重点能力

### 记忆和检索

- 事件驱动 memory write/update/forget 日志。
- current snapshot 表用于快速读取。
- BM25 / FTS5、ONNX embedding、RRF fusion 混合检索。
- 可选 `sqlite-vec` ANN 加速，并保留内存 cosine fallback。
- 保留 namespace 列用于 Pro 前向兼容；Lite 默认仍是单节点、单租户。

### 协议表面

- 通过 `NPS.NWP` 提供 NPS-3 Memory Node host。
- `/.nwm` Neural Web Manifest endpoint。
- `/.schema` Memory Node schema endpoint。
- `POST /api/memory/query` NWP query endpoint。
- 在支持场景下，query response 包含 NCP anchor digest 和 token estimate metadata。

### 身份与信任

- 内置 Ed25519 NIP Mini-CA。
- SQLite-backed CA ledger。
- Agent certificate issue / verify / revoke 命令。
- NPS-3 §8 兼容 CA HTTP routes。
- 通过 `--trusted-issuer` 和 `--ocsp-url` 验证远程 CA。
- NID middleware 支持 `anonymous-allowed`、`writes-required`、`all-required` 三种模式。
- OLS/OIDC 管理员身份、首次 admin setup、login、JWT、SQLite identity stores。

### 工具和集成

- `banyan` 单二进制 CLI，作为 .NET tool 安装。
- Web UI 管理 memory、agent、identity setup 和 CA operations。
- `banyan mcp` stdio MCP server。
- `banyan web` 运行时在 `/mcp` 暴露 Streamable HTTP MCP endpoint。
- 离线 embedder download workflow。

## 安装

```bash
dotnet tool install -g Banyan.Cli --version 1.0.0
```

## 升级

```bash
dotnet tool update -g Banyan.Cli --version 1.0.0
```

升级后确认 tool，并初始化缺失的本地资源：

```bash
banyan --version
banyan embedder download
```

## 首次运行检查清单

```bash
export BANYAN_NIP_CA_PASSPHRASE='your-passphrase'
banyan ca init
banyan init --admin-username admin --admin-password 'change-me-now'
banyan agent issue --id summarizer-01 --cap memory.read,memory.write \
  --key-out ~/.banyan/agents/summarizer-01.key
export BANYAN_EMBEDDER=onnx
banyan web
```

打开 `http://localhost:5180`，用 admin 账号登录。

## Lite 生产使用注意事项

- 常见生产姿态使用 `--nid-auth writes-required`：读公开，写需要鉴权。
- 读写都需要鉴权时使用 `--nid-auth all-required`。
- `BANYAN_NIP_CA_PASSPHRASE` 尽量不要留在 shell history 或 service unit 文件里；优先用宿主 secret manager 注入环境变量。
- 升级前和 CA key rotation 前备份 SQLite 数据目录。
- 使用远程 CA 时，用 `--trusted-issuer` 固定信任根，并确保 OCSP endpoint 可达。
- Lite 应视为单节点部署。需要 tenant isolation 或 fleet topology 时，应走 Pro/Ent 路线，而不是把 Lite 硬扩展成多租户系统。
